# Subtarea 5: Subir los cambios a la rama `develop`

## Descripción del Plan
- Cambiar a la rama `develop`: `git checkout develop`.
- Realizar un `merge fast-forward` desde la rama actual a `develop`: `git merge feature/mi-rama`.
- Subir los cambios a `develop` en el repositorio remoto: `git push origin develop`.
